Kora OS

A few years ago, I started a crazy project. I decided to start writing my own kernel. At the time I didn't have much knowledge of such system so I start learning a lot about system and hardware principes. I already had strong skills in software development but this project really improve my knowledge and allow me to re-learn everything from a different point of view.

Kora is designed to be small and lightweight. I created software with the goal of running only the code's required, nothing more, nothing less. The kernel have been designed as really small bricks, modulable and replacable. My objectif is a clean code easly understandable, piece by piece and not as a big monolith of code.

My OS is still a young pup and I need more time to try some of the countless ideas I have. That's all the fun to create a raw system, everything looks possible... My first objective right now is fixing some issues with signals and multi-threading (stable context-switchng is not that easy), starting my network stack and take a glance at ARM porting.

Features

Listing feature seems like a pointless job in such a program, especialy as this stage, but it's a good way of listing what have been accomplish until now.

  • Virtual memory: Modern kernel manage physical memory while providing application a virtual access to RAM. My kernel can map file, shared memory, as many heap and stack as required and also physical memory at usage of DMA drivers. Private memory pages can also make use of copy-on-write faciliy.
  • Multi-core detection: My kernel is able to detect others CPUs and enable some of there non-default features. The kernel structure itself to run up to 32 CPUs on x86 architecture. However the kernel and especially synchronization systems are not ready to make use of several CPU at the time, not yet.
  • Virtual File System: The virtual filesystem provide an abstraction layer to application to use devices like hard drives, cd-rom and the files stored on them. The VFS is a big piece, for which I decide to give special care as it's the interface with all drivers.

Take a look

If you wish to take a look at KoraOS it's really simple, but not really interresting for now. As soon as I rebuild a nice image with cool interaction I will proposed it. However I take a point to keep it simple. I only need one command to create an turn-key ISO image, with the kernel and basic commands.

The disk image works as a live-cd-rom like most linux distribution. I personnaly test my operating system using VirtualBox or Qemu. I test-it from time to time on real hardware to ensure it work as expected. Running unstable code is not dangerous for your machine, but be-careful of data corruption (avoid if you don't know what your doing).

The code is not always up-to-date with Github repository, but you can still take a look.

Minimum configuration: KoraOS is freaky light! 8 Mb of RAM required, 1 CPU, VGA capable screen (the basic). On debug mode it also push a ton of logs out of COM1 serial port. Make use of IDE disks (HDD, CD), network is comming...

Standard C Library

A doubt that trouble me for quitte some time is the standard library. Should I use an exising one or create my own. As I needed to implement some of the feature for my kernel anyway I decided to give it a try. This gave me more freedom to design the kernel ABI. For the time being, I'm focusing on one standard version at a time. I just completed C89, and starting C95 and C99.

Frequently Asked Questions

Q: Is Kora kernel UNIX-compatible?

A: I think I will defined this kernel as 'mostly compatible'. Because ton of free programs are developed for UNIX system. It would be silly to throw them away, however smoke-os is not meant to be a UNIX system. As such, I put an effort to make such program working on my OS, but compatibility is never garanted and even if it's working, it might not be really optimized against this new platform.

Q: Which application can I use on KoraOS?

A: Sadly not much yet. SmokeOS is a hobby more than a serious systems. I start it for learning purpose and I continue mostly for experimental purpose. Never the less, I hope that some day it will become stable and big enough to be more than this. I wanna try to put an application/packages store for my operating system.
